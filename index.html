What is a Tree?

A tree is a connected structure where each element (called a node) holds a value and can point to child nodes. There's exactly one starting node — the root — and no cycles allowed.

Real-world analogy

Think of a company org chart. The CEO is at the top (root). The CEO has direct reports (children). Each of those has their own reports. Nobody reports to two different managers (no cycles). That's a tree.

Trees · Types

Types of Trees

Binary Tree

Each node has at most 2 children — called left and right. This constraint unlocks powerful search algorithms.

pythonclass BinaryNode:
    def __init__(self, value):
        self.value = value
        self.left  = None   # left child
        self.right = None   # right child

Binary Search Tree (BST)

A binary tree with one extra rule: left child < parent < right child. This makes searching extremely fast — O(log n) in the average case.

Real-world analogy

Think of a dictionary. When you search for "mango", you don't start at "a". You open somewhere in the middle, and decide: do I go left (earlier letters) or right (later letters)? A BST works identically.

pythonclass BST:
    def __init__(self):
        self.root = None

    def insert(self, value):
        if not self.root:
            self.root = BinaryNode(value)
        else:
            self._insert(self.root, value)

    def _insert(self, node, value):
        if value < node.value:          # go LEFT
            if node.left is None:
                node.left = BinaryNode(value)
            else:
                self._insert(node.left, value)
        else:                           # go RIGHT
            if node.right is None:
                node.right = BinaryNode(value)
            else:
                self._insert(node.right, value)

    def search(self, value):
        """Returns True if value exists in BST"""
        node = self.root
        while node:
            if value == node.value: return True
            elif value < node.value: node = node.left
            else: node = node.right
        return False

# Usage
bst = BST()
for n in [10, 5, 15, 3, 7]:
    bst.insert(n)

print(bst.search(7))   # True
print(bst.search(9))   # False
🔍

Search: O(log n)

Each comparison eliminates half the tree. 1 million nodes → at most 20 comparisons.

➕

Insert: O(log n)

Find the correct position using the same left/right logic as search.

⚠️

Worst case: O(n)

If you insert sorted data (1,2,3,4...) the BST becomes a linked list. This is why balanced trees exist.

Trie — The Autocomplete Tree

A trie stores strings character-by-character. Every path from root to a marked node spells a word. It's how your phone keyboard suggests completions.

pythonclass TrieNode:
    def __init__(self):
        self.children = {}      # char → TrieNode
        self.is_end = False   # does a word end here?

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word: str):
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end = True

    def starts_with(self, prefix: str) -> list:
        """Return all words with this prefix"""
        node = self.root
        for char in prefix:
            if char not in node.children:
                return []
            node = node.children[char]
        return self._collect(node, prefix)

    def _collect(self, node, path):
        results = []
        if node.is_end: results.append(path)
        for char, child in node.children.items():
            results.extend(self._collect(child, path + char))
        return results

# Autocomplete demo
trie = Trie()
for w in ["apple", "app", "application", "banana"]:
    trie.insert(w)

print(trie.starts_with("app"))
# ['app', 'apple', 'application']

Trees · Traversal

Traversing Trees: DFS & BFS

Traversal means visiting every node exactly once in a specific order. There are two families:

🔽

DFS — Depth First Search

Go deep before going wide. Uses a stack (or recursion). Three variants: pre-order, in-order, post-order.

↔️

BFS — Breadth First Search

Visit level by level — all nodes at depth 1, then depth 2, etc. Uses a queue.

DFS: Three Orderings

pythondef preorder(node):
    """Root → Left → Right
    Good for: copying a tree, prefix expressions"""
    if not node: return
    print(node.value, end=" ")  # visit BEFORE children
    preorder(node.left)
    preorder(node.right)

def inorder(node):
    """Left → Root → Right
    KEY INSIGHT: On a BST, this gives SORTED order!"""
    if not node: return
    inorder(node.left)
    print(node.value, end=" ")  # visit BETWEEN children
    inorder(node.right)

def postorder(node):
    """Left → Right → Root
    Good for: deleting a tree, computing directory sizes"""
    if not node: return
    postorder(node.left)
    postorder(node.right)
    print(node.value, end=" ")  # visit AFTER children

# If BST contains [3,5,7,10,15], inorder prints: 3 5 7 10 15

BFS — Level Order Traversal

pythonfrom collections import deque

def bfs(root):
    """Visit nodes level by level using a queue"""
    if not root: return

    queue = deque([root])

    while queue:
        node = queue.popleft()          # take from front
        print(node.value, end=" ")

        if node.left:  queue.append(node.left)
        if node.right: queue.append(node.right)

# For the tree A→(B,C)→(D,E,F,G):
# BFS prints: A B C D E F G
# DFS preorder: A B D E C F G
When to use which? Use BFS to find the shortest path or process by level. Use DFS when you need to explore a full path (like finding all paths in a maze), or when you want sorted order from a BST (inorder).

Real Example: Calculate Directory Size

Postorder is perfect here — you must know child sizes before you can compute parent size:

pythonimport os

def get_directory_size(path):
    """Postorder: calculate children first, then sum up"""
    total = 0
    for entry in os.scandir(path):
        if entry.is_file():
            total += entry.stat().st_size     # leaf node
        elif entry.is_dir():
            total += get_directory_size(entry.path)  # recurse
    return total                              # parent sums children

# The file system IS a tree. Recursion is natural here.

Heaps, Balanced Trees & Decision Trees

The Heap

A heap is a binary tree where every parent is larger (max-heap) or smaller (min-heap) than its children. It's stored as an array — no pointers needed!

Real-world analogy

A hospital ER triage queue. The most critical patient is always at the front, regardless of when they arrived. When they leave, the next most critical takes the top spot. That's a min-heap on severity score.

pythonimport heapq

# Python's heapq is a MIN heap (smallest element always first)
tasks = []
heapq.heappush(tasks, (3, "Low priority task"))
heapq.heappush(tasks, (1, "URGENT: server down"))
heapq.heappush(tasks, (2, "Medium priority bug"))

# Always pops the SMALLEST priority number first
while tasks:
    priority, task = heapq.heappop(tasks)
    print(f"[{priority}] {task}")

# Output:
# [1] URGENT: server down
# [2] Medium priority bug
# [3] Low priority task

# Get K largest/smallest — super common interview pattern
nums = [4, 9, 1, 7, 3, 8]
top3 = heapq.nlargest(3, nums)   # [9, 8, 7]

Decision Tree — The Bridge to ML

A decision tree is a regular tree used for classification. Each internal node asks a yes/no question about your data. Leaf nodes are the final prediction. This is one of the most interpretable ML models.

python# A decision tree is just nested if-else — but learned from data
# Here's a manual one for "should I deploy this code?"

class DecisionNode:
    def __init__(self, question=None, yes=None, no=None, answer=None):
        self.question = question   # internal node: a question
        self.yes = yes             # subtree if True
        self.no = no               # subtree if False
        self.answer = answer       # leaf node: final decision

def decide(node, data):
    if node.answer is not None:
        return node.answer          # reached a leaf
    if data[node.question]:
        return decide(node.yes, data)
    return decide(node.no, data)

# Build the tree
tree = DecisionNode(
    question="tests_pass",
    yes=DecisionNode(
        question="code_reviewed",
        yes=DecisionNode(answer="✅ Deploy!"),
        no=DecisionNode(answer="⏳ Wait for review")
    ),
    no=DecisionNode(answer="❌ Fix tests first")
)

print(decide(tree, {"tests_pass": True, "code_reviewed": True}))
# ✅ Deploy!
When sklearn's DecisionTreeClassifier or XGBoost trains, it's building and optimizing exactly this structure — but automatically from data, using metrics like Gini impurity or information gain to decide what questions to ask.

Graphs: Trees Without Rules

A tree is a special case of a graph. A graph is the more general structure — nodes (called vertices) connected by edges, with no restrictions. Cycles are allowed. Multiple paths between nodes are allowed. No root required.

Real-world analogy

A tree is like a strict corporate hierarchy — one boss, clear chain of command. A graph is like LinkedIn — anyone can be connected to anyone, there can be mutual connections, clusters, islands of disconnected people.

BFS and DFS on Graphs

Same algorithms as trees, but now we need to track visited nodes to avoid infinite loops in cycles:

pythonfrom collections import deque

def bfs_graph(graph, start):
    visited = set()
    queue = deque([start])
    visited.add(start)
    order = []

    while queue:
        node = queue.popleft()
        order.append(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return order

def dfs_graph(graph, start, visited=None):
    if visited is None: visited = set()
    visited.add(start)
    result = [start]
    for neighbor in graph[start]:
        if neighbor not in visited:
            result.extend(dfs_graph(graph, neighbor, visited))
    return result

bfs_graph(graph, "Chennai")
# ['Chennai', 'Bangalore', 'Hyderabad', 'Mumbai']

Dijkstra's Algorithm — Shortest Path

The algorithm behind Google Maps. Given weighted edges (distances), find the shortest path from a source to all other nodes.

pythonimport heapq

def dijkstra(graph, start):
    """graph: {node: [(neighbor, weight), ...]}
    Returns: {node: shortest_distance_from_start}"""

    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    heap = [(0, start)]   # (distance, node)

    while heap:
        d, node = heapq.heappop(heap)
        if d > dist[node]: continue   # stale entry

        for neighbor, weight in graph[node]:
            new_dist = d + weight
            if new_dist < dist[neighbor]:
                dist[neighbor] = new_dist
                heapq.heappush(heap, (new_dist, neighbor))

    return dist

# Road network with distances (km)
roads = {
    "A": [("B", 4), ("C", 2)],
    "B": [("C", 1), ("D", 5)],
    "C": [("B", 1), ("D", 8)],
    "D": [],
}
print(dijkstra(roads, "A"))
# {'A': 0, 'B': 3, 'C': 2, 'D': 8}  (A→C→B→D = 2+1+5=8)

GNNs · Core Concept

What is a Graph Neural Network?

A regular neural network takes a fixed-size vector as input (like a row in a CSV). But what if your data is a graph — a molecule, a social network, a knowledge graph? The structure itself carries meaning.

Real-world analogy

Imagine trying to understand a person just from their age, income, and hobbies (a flat vector). Now imagine also knowing who their friends are, and what THEIR friends are like. The graph structure gives you context that numbers alone can't. GNNs capture this.

The Core Idea: Message Passing

Every node in a GNN learns its representation by aggregating information from its neighbors. This is called message passing.

Initialize node features

Each node starts with a feature vector — could be user age, atom type, or any attributes. Shape: [num_nodes × feature_dim]

Pass messages

Each node sends its current feature vector to all its neighbors. Think of it as "broadcasting your state to people around you."

Aggregate

Each node collects messages from all neighbors and combines them — by summing, averaging, or taking the max.

Update

Each node's feature vector is updated by passing the aggregated messages through a small neural network (learned weights).

Repeat K times

After K rounds of message passing, each node has "seen" information from nodes up to K hops away. K=2 means you know your neighbors' neighbors.

python — conceptual message passing (no ML library)import numpy as np

# Simple graph: 4 nodes, each with a 3-dim feature vector
node_features = np.array([
    [1.0, 0.5, 0.2],   # node 0
    [0.3, 0.8, 0.1],   # node 1
    [0.7, 0.2, 0.9],   # node 2
    [0.4, 0.6, 0.3],   # node 3
])

# Adjacency list
neighbors = {0: [1, 2], 1: [0, 3], 2: [0, 3], 3: [1, 2]}

def message_passing_round(features, neighbors):
    """One round: each node averages its neighbors' features"""
    new_features = np.zeros_like(features)
    for node, nbrs in neighbors.items():
        # AGGREGATE: mean of neighbor features
        neighbor_feats = features[nbrs]
        agg = neighbor_feats.mean(axis=0)
        # UPDATE: combine self + neighbors (no learned weights yet)
        new_features[node] = 0.5 * features[node] + 0.5 * agg
    return new_features

# After 1 round, each node "knows" about its neighbors
h1 = message_passing_round(node_features, neighbors)
# After 2 rounds, each node knows about neighbors-of-neighbors
h2 = message_passing_round(h1, neighbors)

print("Before:", node_features[0])
print("After 2 rounds:", h2[0])
The key insight: after K rounds of message passing, node 0's representation reflects the entire K-hop neighborhood around it — the local subgraph structure. This is what GNNs "learn" to do effectively.

GCN, GAT & GraphSAGE

GCN — Graph Convolutional Network (2017)

The foundational GNN. Each node updates its embedding by taking a normalized weighted average of its neighbors. The "convolutional" name is an analogy to image CNNs — both aggregate local context.

The core formula for one GCN layer:

math → then python# H_new = σ( D^(-1/2) · A · D^(-1/2) · H · W )
#
# Where:
#   H   = current node features matrix  [N × F]
#   A   = adjacency matrix (with self-loops) [N × N]
#   D   = degree matrix (diagonal) [N × N]
#   W   = learnable weight matrix [F × F']
#   σ   = activation function (ReLU)
#
# In plain English: aggregate neighbor features (normalized by
# degree so high-degree nodes don't dominate), then transform
# with a learned linear layer.

import numpy as np

def gcn_layer_numpy(A, H, W):
    """Simplified GCN layer (no degree normalization for clarity)"""
    N = A.shape[0]
    A_hat = A + np.eye(N)    # add self-loops
    D = np.diag(A_hat.sum(axis=1))
    D_inv_sqrt = np.diag(1 / np.sqrt(np.diag(D)))
    norm_A = D_inv_sqrt @ A_hat @ D_inv_sqrt

    out = norm_A @ H @ W     # aggregate + transform
    return np.maximum(0, out) # ReLU activation

GAT — Graph Attention Network (2018)

GCN treats all neighbors equally. GAT adds attention: it learns to weight neighbors differently. Some neighbors matter more than others.

Real-world analogy

On Twitter, you follow 500 people — but when forming your opinion on a topic, you weight a domain expert's tweet much higher than a random account. GAT learns these attention weights from data.

python — GAT attention concept# For each pair of connected nodes (i, j):
# 1. Concatenate their transformed features
# 2. Pass through a small learnable vector 'a'
# 3. Apply softmax across all neighbors → attention coefficients

import torch
import torch.nn.functional as F

def attention_coeff(h_i, h_j, W, a):
    """Compute attention weight from node i to node j"""
    Whi = W @ h_i    # transform i's features
    Whj = W @ h_j    # transform j's features
    concat = torch.cat([Whi, Whj])   # [2F]
    e = F.leaky_relu(a @ concat)     # scalar score
    return e   # softmax applied across all neighbors later

# Higher score → node j is more "important" for node i's update
# GAT also uses multi-head attention — run K independent
# attention mechanisms and concatenate results (like Transformers)

GraphSAGE — Inductive Learning (2017)

GCN and GAT are transductive — they can't generalize to new nodes not seen during training. GraphSAGE is inductive: it learns a function to aggregate neighborhood features, so it works on new unseen nodes.

python — GraphSAGE sampling concept# Instead of using ALL neighbors (expensive for large graphs),
# GraphSAGE SAMPLES a fixed number of neighbors

import random

def sample_neighbors(graph, node, k=5):
    """Sample at most k neighbors for scalability"""
    nbrs = graph[node]
    return random.sample(nbrs, min(k, len(nbrs)))

def sage_aggregate(node_features, sampled_neighbors):
    """Mean aggregator — average sampled neighbor features"""
    nbr_feats = [node_features[n] for n in sampled_neighbors]
    return sum(nbr_feats) / len(nbr_feats)

# After aggregation, concatenate with own features and
# apply a linear + normalization step
# Key win: this works on graphs with millions of nodes
# because you never need the full graph in memory at once

GNN Tasks & Training

Three Levels of GNN Predictions

🔵

Node-Level

Classify or predict properties of individual nodes. E.g. "Is this user a bot?" or "What community does this person belong to?"

↔️

Edge-Level

Predict if an edge should exist, or its type. E.g. "Will user A buy product B?" (recommendation systems)

🔮

Graph-Level

Make a prediction about the entire graph. E.g. "Is this molecule toxic?" — pooling all node embeddings into one vector.

Graph Pooling

For graph-level prediction, you need to collapse all node embeddings into one. This is called readout or pooling:

pythonimport torch

def global_mean_pool(node_embeddings):
    """Average all node embeddings → one graph embedding"""
    return node_embeddings.mean(dim=0)

def global_sum_pool(node_embeddings):
    return node_embeddings.sum(dim=0)

def global_max_pool(node_embeddings):
    return node_embeddings.max(dim=0).values

# After pooling: pass graph embedding through a classifier
# graph_embedding → Linear → softmax → is_toxic: yes/no

Semi-Supervised Learning on Graphs

One of GNNs' superpowers: you can have a graph of 10,000 nodes, label only 200 of them, and the GNN learns from the unlabeled nodes' structure too. The labeled nodes "spread" their knowledge through message passing.

python — training loop conceptimport torch
import torch.nn.functional as F

def train(model, data, optimizer):
    model.train()
    optimizer.zero_grad()

    # Forward pass: all nodes get embeddings
    out = model(data.x, data.edge_index)

    # But we only compute loss on LABELED nodes (train_mask)
    # data.train_mask = [True, False, True, ...]
    loss = F.cross_entropy(
        out[data.train_mask],          # predictions for labeled nodes
        data.y[data.train_mask]        # ground truth labels
    )

    loss.backward()   # gradients flow through the entire graph
    optimizer.step()
    return loss.item()

def evaluate(model, data):
    model.eval()
    with torch.no_grad():
        out = model(data.x, data.edge_index)
        pred = out.argmax(dim=1)
        correct = (pred[data.test_mask] == data.y[data.test_mask]).sum()
        acc = correct / data.test_mask.sum()
    return acc.item()

Over-Smoothing: The Main GNN Pitfall

If you stack too many GNN layers, every node's embedding converges to nearly the same value — they've all aggregated information from so far across the graph that they look identical. Typically 2–4 layers is optimal.

python# With K layers, a node aggregates from K-hop neighborhood
# K=1: direct neighbors only
# K=2: neighbors of neighbors (usually best)
# K=6+: nearly the ENTIRE graph → all embeddings look the same
#
# Solutions:
# 1. Limit to 2-4 layers
# 2. Jumping Knowledge Networks (JK-Net) — combine all layer outputs
# 3. DropEdge — randomly drop edges during training
# 4. PairNorm — normalize to maintain diversity

# Edge index: 2 × num_edges tensor (COO format)
# Row 0 = source nodes, Row 1 = target nodes
# Undirected: add both directions!
edge_index = torch.tensor([
    [0, 1, 1, 2, 2, 3],   # sources
    [1, 0, 2, 1, 3, 2],   # targets
], dtype=torch.long)

# Node labels (for classification)
y = torch.tensor([0, 1, 0, 1])

data = Data(x=x, edge_index=edge_index, y=y)
print(data)
# Data(x=[4, 2], edge_index=[2, 6], y=[4])
print(data.num_nodes)   # 4
print(data.num_edges)   # 6

Complete GCN from Scratch with PyG

python — full working GNNimport torch
import torch.nn.functional as F
from torch_geometric.nn import GCNConv
from torch_geometric.datasets import Planetoid

# Download Cora: 2708 papers, 5429 citation edges
# Task: classify each paper into 7 categories
dataset = Planetoid(root='/tmp/Cora', name='Cora')
data = dataset[0]

# ── Define the model ──
class GCN(torch.nn.Module):
    def __init__(self, in_channels, hidden, out_channels):
        super().__init__()
        self.conv1 = GCNConv(in_channels, hidden)
        self.conv2 = GCNConv(hidden, out_channels)

    def forward(self, x, edge_index):
        # Layer 1: aggregate 1-hop neighborhood
        x = self.conv1(x, edge_index)
        x = F.relu(x)
        x = F.dropout(x, p=0.5, training=self.training)
        # Layer 2: aggregate 2-hop neighborhood
        x = self.conv2(x, edge_index)
        return F.log_softmax(x, dim=1)

# ── Setup ──
model = GCN(
    in_channels=dataset.num_features,   # 1433 word features
    hidden=16,
    out_channels=dataset.num_classes    # 7 categories
)
optimizer = torch.optim.Adam(model.parameters(), lr=0.01, weight_decay=5e-4)

# ── Train ──
for epoch in range(200):
    model.train()
    optimizer.zero_grad()
    out = model(data.x, data.edge_index)
    loss = F.nll_loss(out[data.train_mask], data.y[data.train_mask])
    loss.backward()
    optimizer.step()

# ── Evaluate ──
model.eval()
pred = model(data.x, data.edge_index).argmax(dim=1)
acc = (pred[data.test_mask] == data.y[data.test_mask]).float().mean()
print(f"Test Accuracy: {acc:.4f}")  # ~81%

PyG's batching is clever: it merges multiple small graphs into one big disconnected graph, renumbering node indices, and adds a batch attribute telling you which node belongs to which graph. This is why the global_mean_pool function needs the batch vector to pool per-graph.
